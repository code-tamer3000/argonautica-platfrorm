# Эксплуатация: участники, бот, установка PWA

Домен платформы: **https://platform.argonautica-systems.ru**

---

## 1. Добавить участников

Логины участников — их Telegram-ники (без `@`). Список лежит в файле **`users.md`**
в корне репозитория (по одному нику на строку, `@` можно оставить — скрипт его убирает).

1. Открой `users.md` и допиши новые ники (каждый с новой строки):
   ```
   @novyi_uchastnik
   @another_one
   ```
2. На сервере (в корне `/opt/platform`) запусти:
   ```bash
   ./create_prod_users.sh
   ```
   Скрипт создаёт аккаунты (идемпотентно — уже существующие пропускаются), для каждого
   заводит личный канал и печатает список `логин⇥пароль` в файл `credentials_<дата>.txt`.

> Пароли можно раздать вручную из этого файла **или** через Telegram-бота (см. ниже) —
> тогда раздавать ничего не нужно, участник получает пароль сам.

Файл `credentials_*.txt` после раздачи удали: `shred -u credentials_*.txt`.

## 2. Telegram-бот выдачи паролей

Участник пишет боту в личку → бот проверяет, что его Telegram-`@username` совпадает с
логином на платформе → выдаёт свежий одноразовый пароль и инструкцию по установке.
Пароль в БД хранится только хешем, поэтому бот его **пере-выдаёт** (не «достаёт»);
при первом входе система попросит сменить пароль.

> Важно: бот только выдаёт доступ уже заведённым людям. Сначала аккаунт нужно создать
> шагом 1 (`create_prod_users.sh`). Иначе бот ответит «не нашёл участника с таким логином».

Бот работает по **MTProto через Telethon** (не HTTP Bot API): на RU-хостинге IP
Telegram заблокированы, поэтому `api.telegram.org` недоступен, а MTProxy до Telegram
достучаться может. Поэтому нужен MTProxy (Server/Port/Secret) и api_id/api_hash.

### Настройка (один раз)

1. В Telegram у **@BotFather** создай бота (`/newbot`), получи токен.
2. На https://my.telegram.org → **API development tools** получи `api_id` и `api_hash`.
3. Возьми MTProxy (Server / Port / Secret) — тот, что показывает панель прокси.
4. На сервере в `.env` пропиши:
   ```
   TELEGRAM_BOT_TOKEN=123456:ABC-DEF...
   TELEGRAM_API_ID=1234567
   TELEGRAM_API_HASH=abcdef0123456789abcdef0123456789
   PLATFORM_URL=https://platform.argonautica-systems.ru
   # MTProxy из панели прокси:
   TELEGRAM_MTPROXY_SERVER=gt-yxorp.kvn-npv.com
   TELEGRAM_MTPROXY_PORT=8443
   TELEGRAM_MTPROXY_SECRET=ee6b696e6f...
   ```
5. Подними сервис бота:
   ```bash
   docker compose -f docker/docker-compose.prod.yml --env-file .env up -d bot
   ```
4. Логи бота:
   ```bash
   docker compose -f docker/docker-compose.prod.yml --env-file .env logs -f bot
   ```

Бот использует тот же образ, что backend (собирается при деплое). После правок кода бота
пересобрать и перезапустить: `... up -d --build bot`.

### Что получает участник

Бот присылает: ссылку на платформу, логин, пароль и инструкцию по установке (см. ниже).

## 3. Установка PWA на телефон

Приложение ставится прямо из браузера (нативных приложений в сторах нет).

**iPhone / iPad (Safari):**
1. Открой **https://platform.argonautica-systems.ru**
2. Нажми «Поделиться» ⬆️ (квадрат со стрелкой внизу).
3. Выбери «На экран «Домой»» → «Добавить».

**Android (Chrome):**
1. Открой **https://platform.argonautica-systems.ru**
2. Меню ⋮ (три точки справа вверху).
3. «Установить приложение» (или «Добавить на главный экран»).

После установки иконка появится на рабочем столе — приложение открывается на весь экран,
как обычное. Первый вход — по логину/паролю, затем система попросит сменить пароль.
